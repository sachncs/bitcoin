## Responsibilities

- Review and merge pull requests.
- Triage and respond to issues.
- Release new versions to PyPI.
- Maintain CI/CD pipelines and infrastructure.
- Enforce the Code of Conduct.
- Set project roadmap and priorities.

## Becoming a Maintainer

1. Demonstrate sustained, high-quality contributions (typically 6+ months of
   active participation).
2. Be nominated by an existing maintainer.
3. Receive consensus approval from existing maintainers.
4. Accept the responsibilities and time commitment.

Maintainers who are inactive for 6+ months may be moved to emeritus status.
